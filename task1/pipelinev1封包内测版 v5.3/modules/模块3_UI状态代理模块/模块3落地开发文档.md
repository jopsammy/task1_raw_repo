## 开发背景

### 项目背景

在Streamlit应用中，页面刷新会导致状态丢失。为了解决这个问题，需要一个统一的状态管理模块，负责：

1. 管理Streamlit session_state与底层数据的同步
2. 跟踪Pipeline执行状态、进度和日志
3. 提供状态持久化功能
4. 管理提示词调试面板的状态

### 技术选型

- **状态存储**: Streamlit session_state（会话内）+ JSON文件（持久化）
- **单例模式**: 确保全局只有一个StateProxy实例
- **Streamlit兼容**: 支持可选导入，模块可在非Streamlit环境中使用

## 架构设计

### 类结构

```
StateProxy (主类)
  ├── _init_session_defaults() - 初始化session_state默认值
  ├── load_project() - 加载项目
  ├── save_project() - 保存项目
  ├── mark_dirty() - 标记脏数据
  ├── update_pipeline_status() - 更新Pipeline状态
  ├── add_pipeline_log() - 添加Pipeline日志
  ├── get_pipeline_status() - 获取Pipeline状态
  ├── store_pipeline_result() - 存储Pipeline结果
  ├── get_pipeline_result() - 获取Pipeline结果
  ├── update_prompt_debug() - 更新提示词调试状态
  ├── get_prompt_debug_state() - 获取提示词调试状态
  ├── persist_state() - 持久化状态
  └── load_persisted_state() - 加载持久化状态

PipelineStatus (枚举)
  ├── IDLE
  ├── RUNNING
  ├── PAUSED
  ├── COMPLETED
  └── FAILED
```
