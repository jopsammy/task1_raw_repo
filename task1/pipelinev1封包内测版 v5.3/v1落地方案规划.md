
# 需求结构化分析工具 v1 落地方案规划

&gt; 基于AC锚点契约式人机协同开发范式，构建专业级需求结构化分析工具

---

## 一、项目背景与目标

### 1.1 项目背景

在AI开发时代，需求分析是整个软件开发生命周期的第一道关键环节。传统的需求文档往往是自然语言表述，存在**歧义性高、结构松散、可追溯性差、难以自动化处理**等问题。

本项目旨在构建一个**专业级需求结构化分析工具**，通过AC范式的锚点与契约机制，将自然语言需求转化为**结构化、可追溯、可校验、可机器处理**的标准格式，为后续的AI辅助开发、自动化测试、项目管理提供坚实基础。

### 1.2 核心目标

| 目标维度 | 具体目标 |
|---------|---------|
| **功能目标** | 支持需求输入、结构化拆解、锚点绑定、契约生成、多格式导出 |
| **质量目标** | 结构化准确率&gt;90%，可追溯性100%，容错与降级机制完善 |
| **性能目标** | 单次分析响应时间&lt;30秒，支持10万级需求条目管理 |
| **扩展性目标** | 模块化设计，支持提示词工程模块独立迭代 |

### 1.3 非功能约束

- **技术栈**：Python + Streamlit（UI）+ JSON（存储）
- **双入口设计**：同时提供Streamlit可视化界面和CLI命令行入口
- **LLM友好**：所有代码包含中文docstring，模块化设计适配LLM上下文窗口
- **契约优先**：所有数据结构、接口规范预先定义为契约

### 2.1 标准目录树

```text
v1/
├── config/                          # 配置文件目录
│   ├── config.json                  # 主配置文件（不提交版本控制）
│   └── config_template.json         # 配置模板（提交版本控制）
```

|---------|-------|---------|--------|

### 3.2 模块3_提示词工程模块（核心突出）

| 维度 | 详细说明 |
|-----|---------|
| **模块定位** | 整个系统的「智能大脑」，所有LLM调用的提示词均由本模块统一管理和生成 |
| **核心价值** | 提示词与业务逻辑解耦，支持独立迭代优化，无需修改业务代码即可提升分析质量 |
| **核心职责** | 1. 提示词模板库管理&lt;br&gt;2. 提示词动态生成（上下文注入）&lt;br&gt;3. 提示词版本控制&lt;br&gt;4. 提示词A/B测试支持 |
| **依赖模块** | 模块1（数据锚点模块）- 用于存储提示词模板 |
| **输出** | 结构化的提示词对象（含系统提示词、用户提示词、参数等） |
| **文件位置** | `modules/模块3_提示词工程模块/` |
| **关键接口** | `generate_prompt(template_id: str, context: dict) -&gt; dict`&lt;br&gt;`get_template(template_id: str) -&gt; dict`&lt;br&gt;`list_templates(category: str) -&gt; list` |

| **模块5_导出与报告模块** | 多格式导出、报告生成 | 模块1 | 导出文件（Markdown/JSON/Excel） | `modules/模块5_导出与报告模块/` | `export_requirement(req_id: str, format: str) -&gt; str`&lt;br&gt;`generate_report(req_id: str) -&gt; str` |

| Streamlit状态管理复杂 | 中 | 中 | 遵循Streamlit规范、固定key、使用st.session_state |
