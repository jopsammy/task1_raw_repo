### 2.3 x-seq 包装配对法则（统一公式 🆕）

**统一 wrapping 公式**（`route_g_probe.py`，2055/2055 全量验证通过）：

```
Xw = pair_seq([1] + X) = [1+x1] + pair_seq(X[1:])

kw   = 1 + tail_R_k
bobw = tail_R_bob + (sum_X - x1)

其中：tail_R = pair_seq(X[1:])，sum_X = Σ X[i]
```

**关键发现**：

* sum\_X ≠ n\_pairs（如 `()(())` 的 X=\[1,2]，sum=3 > n=2，必须独立追踪）

* x1≥2 时 wrapping **不是**恒等变换（455/1429 反例，之前的认知是错误的）

* 公式对 x1=1 和 x1≥2 均成立（统一有效）

**路线 G 实验发现的已废弃假设**：

* ~~x1≥2 → wrapping 是恒等变换~~ ← 455 反例

* ~~bob 存在 wrapping 线性不变量~~ ← 搜索证伪

* ~~sum\_X = n\_pairs~~ ← 对部分 Dyck words 不成立

### 2.4 wrapping 修正参数（已过时——仅覆盖前导连续1，不覆盖内部1）

> **⚠️ 2025-05-21 重大发现**：以下表格仅对「前导连续1」（c）正确。配对法则 `pair_seq` 作用于**整个 x-sequence**，内部1也会配对。例如 `X=[2,1,1]` 的 wrapping 产生 `X'=[3,2]`（内部 `(1,1)→2`），而非 naive `[3,1,1]`。**仅按 c 分类的 DP 从根本上不完整。**

| c（前导连续 1 个数） | dk = Δk | db\_head（领先部分 bob 修正） | 适用范围  |
| ------------ | ------- | --------------------- | ----- |
| 2            | 1       | 0                     | r>0   |
| 3            | 1       | 1                     | r≥0   |
| 4 (r=0)      | 1       | 2                     | 仅 r=0 |
| 4 (r>0)      | 2       | 2                     | 仅 r>0 |
| 5            | 2       | 4                     | r≥0   |
| 6 (r=0)      | 2       | 6                     | 仅 r=0 |
| 6 (r>0)      | 3       | 6                     | 仅 r>0 |
| 奇 c=2p+1     | p       | p(p+1)                | r≥0   |

| 偶 c=2p (r=0) | p-1     | p(p-1)                | 仅 r=0 |

> **修正**：偶c的 db\_head = p(p-1) = (c/2)(c/2-1)，原文档的 p² 是早期错误推导。偶c dk 在 r=0/r>0 下不同。
>
> 修正因子：`b^{-dk·L - dk·r - db_head}`。per-word 验证通过（n≤4）。n≥5 因内部1修正缺失而失败。

## 三、已探索的 DP 路线与问题

### 3.1 路线 1：(A)B 分解（已放弃）

**思路**：用标准 Catalan 递推 `F[n] = Σ f(k,n)·F[k]·F[n-1-k]`。

**问题**：(A) wrapping 时 x-seq 跨步合并（如 `(()(()))` 的 x-seq=\[2,2] 而非 \[2,1,1]），步骤边界不遵循匹配括号边界，导致 (A)B 框架下 bob 递推不可解。

### 3.2 路线 2：primitive-concat DP（当前实现）

**思路**：用 Dyck word 的 primitive 分解替代 (A)B 分解。定义生成函数：

```
G_n(L) = Σ a^{alice(s)} · b^{bob(s) + k(s)·L}
P_n(L) = Σ_{primitive} a^{alice(s)} · b^{bob(s) + k(s)·L}
```

**concat 递推**：

```
G_n(L) = Σ_{m=1..n} a^{m(n-m)} · P_m(L+n-m) · G_{n-m}(L)
```

其中 P\_1(L) = a·b^L，对于 m≥2：`P_m(L) = a^{2m-1} · G_{m-1}(L) + wrapping corrections`。

**wrapping correction（对 c≥2 连续前导 1）**：

```
P_n(L) = a^{2n-1}·G_{n-1}(L) + a^{2n-1}·Σ_{c=2}^{n-1} δ_c(L)·G0_r(L)
```

其中 δ\_c(L) = (b^{-dk·L - dk·r - db\_head} - 1) × base\_factor。

**当前验证状态**：

* n=1..4：全对（brute-force 验证）

* n=5..8：G0 聚合存在偏差，P\[n]\[0] 值错

**根因分析（n=5 案例）**：

* c=2, r=2：修正项 delta=771751937，G0\[2]\[0]=16 ✓

* c=4, r=0：修正项 delta=658505729，G0\[0]\[0]=1 ✓

* P\_naive=345505791，P\_corrected=777519104，P\_expected=607649792

* **总偏差 = 828375041。问题在于两个修正项同时作用于同一个 P\_n\[L] 时产生交叉污染——c=4 修正覆盖了 c=2 修正已处理的词的一种子集重叠**

**本质困难**：wrapping correction 公式对单个词正确，但 DP 状态 (n,L) 无法区分同一 (c,r) 分组内不同 x-seq 尾模式的词，导致修正因子被错误聚合。

### 2.5 树DP + wrapping 已验证公式（2025-05-21 树DP探索）

**树分解**：Dyck word ↔ ordered rooted tree。Tree = root + \[T₁,...,Tₘ]。Dyck = P₁·P₂·...·Pₘ，其中 Pᵢ = (Tᵢ)。

**concat 公式**：✅ n≤8 全量验证。
**包装 alice**：`alice((A)) = alice(A) + 2·|A| + 1` ✅ 全量验证。

**包装 bob/k (per-word 正确)**：

* **x₁ ≥ 2**：`k((A))=1+kt`，`bob((A))=bobt+(|A|-x₁)` ✅ n≤8

* **x₁ = 1**：`k((A))=1+k_R(Q)`，`bob((A))=bob_R(Q)+(|A|-1)` ✅ n≤8

* 其中 kt,bobt = pair\_seq(tail)，k\_R,bob\_R = pair\_seq(full X)

**树DP 验证结果**：

* n≤4：完全匹配 brute-force ✅

* n=5：42词中41词匹配，仅 `(()(()()))` 失败（内部1配对 `pair_seq≠x_seq`）

* 失败词：T₀=`()(()())`（X=\[1,2,1]），pair\_seq → \[2,2,1] ≠ x\_seq → \[2,3]

### 2.6 `pair_seq([1]++X(A)) ≠ X((A))` 的根本原因

**实验验证**：对 A=`"()(()())"`（X=\[1,2,1]），

* `pair_seq([1,1,2,1]) = [2,2,1]`，bob=4, k=3

* `x_seq_and_bob("(()(()()))") = [2,3]`，bob=3, k=2

**根因**：x-seq 算法在 wrapping 时受外括号 +1 高度偏差影响：

* `x'₁ = x₁(A) + 1`（始终成立）

* `x'ⱼ = max{t: h_A(2l_j-1+t) = t-1}`（j≥2），其中 `l_j = Σ_{i<j} x'_i`

* 标准 x-seq：`xⱼ = max{t: h_A(2l_j+t) = t}`

**差异**：条件从 `h = t` 变为 `h = t-1`，使外括号的 +1 高度产生额外合并。

**结论**：

1. `pair_seq` 仅在 X(A) 由 `≥2` 组成时与 `x_seq_and_bob` 一致
2. 任何出现 `1` 的情况下，wrapping 的 x-seq 取决于**完整高度函数**，不能从 Augmented x-sequence 的简单配对还原
3. **wrapping 变换由** **`(x₁, last, k, bob)`** **完全确定**（n≤6 数据验证）

### 2.7 wrapping 变换的不可约性

对 n=5-6，wrapping 变换依赖于 (x₁, last, k, bob) 四元组，而非更少参数：

* (x₁, last) 仅：n=5 有 1 个歧义键，n=6 有 3 个

* (k, bob) 仅：无法确定 wrapping 结果

**对 DP 设计的影响**：

* 需要至少 4D 状态才能表示完整的 wrapping 变换

* 4D DP (n, x₁, last, k) 空间 O(N⁴) ≈ 6.25×10¹⁴，完全不可行

* 需要用生成函数/卷积式方法压缩这些维度

通过枚举 n=1..6 全部 Dyck word 发现：

**pairing 法则**（`pair_seq`）：对序列 `seq`，从 i=0 扫描：

* 若 `seq[i]=1` 且 `i+1<|seq|`：输出 `1+seq[i+1]`，i←i+2

* 否则：输出 `seq[i]`，i←i+1

`X((A)) = pair_seq([1] ++ X(A))`。

**关键洞察**：pairing 不限于"前导1合并外括号"。**任何位置的1**（只要后面还有元素）都会与后继配对！

| X(A) 模式           | 实例         | X' = pair\_seq(\[1]+X) | naive X'   | dk | dbob |
| ----------------- | ---------- | ---------------------- | ---------- | -- | ---- |
| \[v, 1] v≥2       | \[2,1]     | \[3,1]                 | \[3,1]     | 0  | 0    |
| \[v, 1, 1] v≥2    | \[2,1,1]   | \[3,2]                 | \[3,1,1]   | 1  | 1    |
| \[v, 1, 1, 1] v≥2 | \[2,1,1,1] | \[3,2,1]               | \[3,1,1,1] | 1  | 2    |
| \[1, v, 1] v≥2    | \[1,2,1]   | \[2,2,1]               | \[2,2,1]   | 0  | 0    |
| \[1, v, 1, 1] v≥2 | \[1,2,1,1] | \[2,2,1]               | \[2,2,1,1] | 1  | 2    |
| \[2, 1, 2]        | \[2,1,2]   | \[3,2]                 | \[3,1,2]   | 1  | 1    |

**结论**：

1. 只有所有元素都 ≥2 的 x-sequence 才不需要修正
2. 任意位置的连续1都会配对，与 leading 1 无关
3. 仅按 c（前导1个数）分类的 DP 从根本上不完整
4. 需要 track x-sequence 中"1的分布"，至少需要知道首位是否为1

## 四、三重阻塞问题

### P1：正确性（GATE-1）

* **现象**：n=1..4 全对，n≥5 wrapping correction 聚合偏差

* **根因**：2D DP 状态 (n,L) 语义粒度过粗，同一 (c,r) 分组内多个 wrapping 修正相互干扰

* **数学本质**：wrapping 修正依赖 G0\[r]\[L] 的历史聚合——而该聚合本身已被之前的 wrapping 修正间接修改

* **可能方向**：3D DP (n, L, a₁) 追踪首步长度；或改为"wrapping→concat"拓扑序（先算所有 P 再算 G）

### P2：时间（GATE-2）

* **现象**：N=500 5s → N=5000 5000s（250× 超限）

* **根因**：concat 递推 `G_n(L) = Σ_m a^{m(n-m)}·P_m(L+n-m)·G_{n-m}(L)` 中 `a^{m(n-m)}` 同时耦合 m 和 n

* **加速尝试**：

  * ✗ 分离 `a^{mn-m²} = (a^n)^m·a^{-m²}` → 仍依赖 n

  * ✗ NTT 卷积 → L-shift `P_m(L+n-m)` 非标准卷积

  * ✗ 对角线 DP（固定 d=n+L）→ G\_t 的 m 依赖未消除

* **可能方向**：生成函数法 `F(x,y) = Σ a^{alice} b^{bob} x^n y^L` → 若存在闭式或微分方程则可 O(N²)

### P3：内存（GATE-3）

* **现象**：3 个三角数组 \~1000MB，限制 600MB

* **缓解**：可降为 2 数组（P+G）≈ 700MB，仍超限但不致命；可降为 1 数组 + 即时计算 ≈ 350MB

* **非致命但需解决**：内存优化无法解救 P2（时间）

### 6.1 L(i) 序列表示 — 路线 K 的数学基石（2025-05-22）✅

**来源**：`blind-spot-analysis-note.md` 独立分析。

**定义**：

* `P(i)` = 第 i 个右括号在字符串中的位置（1-based）

* `L(i) = P(i) - i` — 第 i 个右括号左侧的左括号总数

* `L(i) - i` = 遇到第 i 个右括号时的当前高度

**三个等价公式**（n≤5 全量验证通过）：

1. `alice = Σ_{i=1..n} L(i)` — **纯线性求和**，不需要任何分解

### 11.3 (area, bounce) 分布特征

* area 范围 \[0, n(n-1)/2]，bounce 范围 \[0, n(n-1)/2]

* 分布不是均匀的：最大 bounce 对应 area=0（全嵌套词 `(((...)))`），最大 area 对应 bounce=0（平坦词 `()()...()`）

## 十二、DP 数学重构分析（2025-05-24）🔴

### 12.1 转移方程展开

从 L(i) DP 出发，消去 y 维度后的核心转移：

$$dp\[x\_p]\[x\_p+u] += S\[x]\[x\_p] \cdot b^{n-x} \cdot a^{x\_p^2 - x\cdot x\_p + u} \cdot \binom{x\_p-x-2+u}{x\_p-x-2}\_a$$

其中 `S[x][xp] = Σ_{y=x}^{xp} dp[x][y]`。

### 12.2 a^{d·xp} 耦合的结构性

令 `d = xp - x`。核心耦合项 `a^{-x·xp}` 将源 x 和目标 xp 乘法性耦合在指数中——这是**交叉项（cross term）**，不是卷积形式。

尝试的仿射变换 `dp'[x][y] = dp[x][y] · a^{-f(x,y)}`：

| f(x,y) | 结果                |
| ------ | ----------------- |
| f=0    | 耦合项 a^{-x·xp} 仍存在 |
| f=x²   | 产生 -x·d 耦合        |
| f=xy   | 产生 x²+xp²-x·xp 交叉 |

**结论**：不存在仿射变换消除 a^{-x·xp} 耦合。这解释了为什么所有 O(N³)→O(N²) 降阶尝试均失败。

### 12.3 Route D 与 Route L 的等价性

| 路径      | "便宜"部分               | "贵"部分                                         |
| ------- | -------------------- | --------------------------------------------- |
| L(i) DP | H = q-binomial O(N²) | (x,xp,u) 三重循环 O(N³)                           |
| Route D | F 递推 O(N²)           | H\[k]\[L] = Σ\_{kw} C\[k]\[kw]·b^{kw·L} O(N³) |

两条路径收敛到同一个 O(N³) 瓶颈：都需要对 k→N 的 wrapping 分布进行加权聚合。

#### 7.0 范式转换：为什么路线 A\~K 全部失败

**一句话根因**：

> bob定义在x-seq轴上（Σ(j-1)·xⱼ），所有路线A\~K在Dyck词结构上做分解（树/primitive/wrapping）。两种分解切在不同位置——这是结构性不兼容。

路线 A\~K 的共同死因：

```
旧范式：Dyck词 → 分解(树/primitive/(A)B) → 处理wrapping → 组装
                        ↑                    ↑
                   alice完美             bob需要修正
                   (树结构)             (x-seq跨结构)
```

x-seq分解的块边界 ≠ 树分解的节点边界 ≠ primitive分解的块边界。无论怎么换分解轴，只要在Dyck词结构上做分解，bob的跨结构问题就绕不开。

#### 7.1 L(i)序列的核心公式（n=1..7全量验证通过）

```
P(i) = 第i个右括号在字符串中的位置（1-based）
L(i) = P(i) - i
```

**Alice**：alice = Σ\_{i=1..n} L(i) — 纯线性求和，完全不需要任何分解！

**Bob**：跳跃链 l₀=0, lⱼ₊₁=L(lⱼ+1)。bob = Σ\_{j=1}^{k-1} (n - lⱼ) — 每跳贡献独立于步序j！

**关键洞察**：

* 每个跳跃lⱼ对bob的贡献是(n - lⱼ)，仅依赖lⱼ和n，完全不依赖"这是第几步"（即不依赖j）

* 这意味着状态设计中可以**消去k（步数）维度**

* 跳跃链 lⱼ₊₁ = L(lⱼ+1) 是一阶马尔可夫——下一个跳跃仅取决于当前跳跃点的L值

#### 7.2 DP框架

**状态**：dp\[x]\[y] — 以x为跳跃点且L(x)=y的所有部分序列的加权和。

**跳跃链结构**：

```
l₀=0 → l₁=L(1) → l₂=L(l₁+1) → ... → lₖ=n
         ↑bob+=n-l₁   ↑bob+=n-l₂
```

**转移**：

1. 当前跳跃点x，L(x)=y
2. 下一跳跃触发位置：y+1
3. 位置x+1..y的L值：单调非降序列，仅贡献alice（a^{ΣL}）
4. L(y+1)=y'决定下一个跳跃点
5. bob贡献：n-x（前一个跳跃点的贡献）

**中间段组合计数**：位置x+1..y共(y-x)个位置，L值单调非降从≥y到≤y'，每个L(i)贡献a^{L(i)}。这是有界非降序列的带权和。

#### 7.3 算法复杂度

* 状态空间：O(N²)（x∈\[0,n], y∈\[x,n]）

* 直接转移：每状态O(N)枚举y'，总O(N³)

* 目标：前缀和/对角线优化至O(N²)

* 内存：O(N²) ≈ 200MB（N=5000时约1250万个状态，每状态4字节≈50MB，加表≈200MB），在600MB内

#### 7.4 与旧路线的本质区别

| 维度       | 旧范式(A\~K)               | 新范式(L)                  |
| -------- | ----------------------- | ----------------------- |
| 表示       | Dyck词 + 分解(树/primitive) | L(i)序列(1:1对应)           |

## 八、全局反思：为什么13条路线全部卡在wrapping（2025-05-23 终判）

### 8.1 不是路线选择问题

全部路线 A\~K（11条）加上路线 L（DP框架）和 GF 路线（生成函数框架），共 13 条路线，5 种完全不同的分解范式（树分解、x-seq分解、c-based递推、L(i)序列DP、生成函数），无一例外全部在 wrapping 步骤受阻。这不是巧合。

### 8.2 wrapping 的数学本性

wrapping `A → (A)` 需要知道 A 的**完整高度函数** h\_A(t) 在 t∈\[0,2|A|] 上的全部取值。高度函数的信息维度是 O(2^k)，而任何 O(1) 或 O(k^c) 维度的 DP 状态都无法无损存储这个信息。

因此：

* 所有基于"局部参数 + DP"的路线（A\~H, K）都因为状态粒度不足以区分不同 height function 而失败

* 路线 I 试图用 x-seq 作为代理参数，但 x-seq 本身丢失了高度信息（多对一映射）

* 路线 L 将 wrapping 嵌入 H\[k]\[L] 的全体求和，信息量没有丢失，但 H 本身需要 O(k³) 轨道来存储

* GF 路线将 wrapping 嵌入 C\[k]\[kw] 的全体求和，同样需要 O(k³) 轨道

### 8.3 问题的最简形式

经全部路线探索后，原问题被等价归约为一个核心数学对象：

**q-Dyck 的 wrapping 生成函数**：
$$C\[k]\[kw] = \sum\_{A: |A|=k,\ k((A))=kw} a^{\text{alice}(A)} \cdot b^{\text{bob}((A))}$$

求 C\[k]\[kw] 的递推关系或生成函数方程，等价于求解原问题。

### 8.4 GF框架（已建立）

$$F(x) = 1 + \sum\_{k \ge 0,\ kw} C\[k]\[kw] \cdot a^{2k+3} \cdot x^{k+2} \cdot F(a^{k+2} \cdot b^{kw} \cdot x)$$

此方程为 q-模拟（q-analog）生成函数的特征形式。MOD = 998244353 = 119×2²³+1 支持 NTT 加速。

### 8.5 下一步

**攻克「q-Dyck 的 wrapping 生成函数」**：为 C\[k]\[kw] 建立独立的递推关系或生成函数方程。

候选路径：

* 从已枚举的 k≤14、120 个轨道数据中反推 C\[k]\[kw] 的递推模式

* 为 C(y,z) = Σ C\[k]\[kw]·y^k·z^{kw} 建立独立的 GF 方程

* 利用 NTT 设计 Newton 迭代直接提取 F\[n] 系数

### 8.6 已认识到的结构事实

* alice = Σ L(i) — 纯线性，零分解，O(1)

* bob = Σ (n - lⱼ) — 跳跃链求和，一阶马尔可夫，O(1)

* concat — 完美可加，不需要修正

* wrapping — 等价于完整高度函数，信息维度 O(2^k)，不可低维压缩

**整道题的非平凡部分浓缩在 wrapping 这单一操作上。**

### 8.7 C\[k]\[kw] 递推探索（2025-05-23 终判）

对 k=0..14 的 C\[k]\[kw] 全量数据进行了三项系统测试：

1. **c-based 分解（个体词级）**：c>0 且 rest\_kw>0 时出现歧义——同 (c, bw\_rest, kw\_rest) 可对应多种不同真实 wrapping 结果。wrapping 不是 c + rest 参数的函数。

2. **c-based 分解（orbit 聚合级）**：即使将 rest 词按 (bw\_rest, kw\_rest) orbit 聚合，同一 orbit 内的不同词仍可 wrapping 到不同目标 orbit。反例：c=2, rest\_orbit=(0,1) 产生 7 种不同 (bw,kw)。

3. **行间差分/比值测试**：无常数比率或简单代数关系。

**结论**：C\[k]\[kw] 不存在简单递推。wrapping 的全局性在 orbit 级别同样成立。问题内核（q-Dyck wrapping 生成函数）的解析结构尚未被破译。

### 九、q-二项式分析（2025-05-23 补录）

针对 `blind-spot-analysis-note0523v2.md` 提出的 q-二项式 + NTT 优化路径进行了独立验证（`verify_qbinomial_claim.py`）：

**确认的事实**：

* ✅ H\[d]\[u] = binom(d+u, d)\_a。路线 L 的 DP 已经使用了 q-二项式系数，由 `precompute_H_and_pow` 在 O(N²) 内计算。

* ✅ 对 N=10, a=2 全量 121 项，H 与 q-binom 完全一致。

**证伪的论断**：

* ❌ blind-spot 的简化 DP 转移 `dp[y][z] = b^{n-y} · Σ_x dp[x][y] · a^{y(y-x)} · binom(z-x-1, z-y)_a` 不成立。

  * 状态表示 dp\[y]\[z] 仅追踪跳跃点序列，丢失了 L(jump\_point) 信息。

  * L(x') 不可从 x' 推导（x' = P(x')? No, x' is a jump point value, L(x') depends on string position）。

  * binom(z-x-1, z-y)\_a 不依赖 yp=L(xp)，而验证过的 H 明确依赖 yp。

* ❌ NTT 加速不适用：H\[d-2]\[yp-xp] 是对角线操作（仅依赖 yp-xp 差值），非交叉相关/卷积形式。固定 d 和 xp 时 dp\[xp]\[\*] 通过逐点乘法计算。

**开放问题**：

* H 的 q-二项式身份可被利用——例如推导 H 的生成函数闭式或组合恒等式

* 生成函数 G\_n(y) = y · Σ\_s A\[s] · a^{s(n-s)} · G\_{n-s}(b^s y) 中的非均匀缩放 b^s 是当前框架降阶的主要障碍

* q-Catalan 标准形式 C(z,q) = 1 + z·C(z,q)·C(zq,q) 不含非均匀缩放，暗示此题可能需特殊处理

#### 10.1 方向 A：yp 前缀和（`route_l_dp_v3.py`）

与方向 B 同构（系数提取 = NTT 卷积），B 已验证此路径不可行。已排除。

## 十三、Bergeron/Garsia-Haglund 递推验证（2025-05-24）🟢

### 13.1 递推公式

$H\_{n,k}(q,t)$ 递推（1 ≤ k ≤ n）：

$$H\_{n,n}=q^{\binom{n}{2}},\quad H\_{n,k}=t^{n-k}q^{\binom{k}{2}}\sum\_{r=1}^{n-k} \binom{r+k-1}{r}_q H_{n-k,,r}$$

$$C\_n(q,t)=t^{-n}H\_{n+1,1}(q,t),\quad F\[n]=a^{n(n+1)/2}C\_n(a,b)$$

### 13.2 验证结果

n=1..8 四组 (a,b) 全量验证通过。验证脚本：`verify_bergeron.py`。
N=500 实测 3.89s（vs L(i) DP v3 的 3.49s）。

### 13.3 三条路径同归 O(N³/6)

L(i) DP (O(N³/2))、Route D (O(N³))、Haglund 递推 (O(N³/6))——三条独立路径收敛于同一瓶颈：需对所有 k,r 枚举 q-binomial 加权求和。

## 十五、(q,t)-Catalan 递推全貌（2025-05-24）

经过系统性验证的全部 (q,t)-Catalan 相关递推：

| 递推                      | 复杂度     | 适用性           |
| ----------------------- | ------- | ------------- |
| Garsia-Haglund H\_{n,k} | O(N³/6) | 一般 (a,b)      |
| Garsia b=1 递推           | O(N²)   | 仅 b=1 或 a=1   |
| b=a⁻¹ 闭合式               | O(N)    | 仅 b=a⁻¹ mod M |

**对称性**：C\_n(a,b) = C\_n(b,a)（Garsia-Haiman 96）。

**结论**：Garsia-Haglund O(N³/6) 递推是唯一对一般 (a,b) 成立的递推。该递推与 L(i) DP (O(N³/2))、Route D (O(N³)) 数学上等价，收敛于同一 q-binomial 加权求和瓶颈。三条独立路径的收敛强化了 q-binomial 加权求和不可降阶的结构事实。

### 16.3 v8/v8b 失败分析

**v8**：`#include <immintrin.h>` + `_mm256_*` intrinsic 直接写在 `main` 中。判题机不带 `-mavx2` 编译 → 20 errors。

**v8b**：用 `__attribute__((target("avx2")))` 隔离 AVX2 代码到独立函数 + `__builtin_cpu_supports("avx2")` 运行时检测。判题机编译通过，但 **0/19 用例答案错误** —— target 属性在旧编译器上产生错误代码。

**结论**：判题机编译器不支持任何形式的 AVX。永久禁用 `<immintrin.h>`、`__attribute__((target))`、`-mavx2`、`-march=native`（后者可能自动启用 AVX2）。

### 16.4 v9 架构

纯标量 32路 ILP 展开 + `__builtin_prefetch` + `__int128` 聚合。无任何 SIMD intrinsic 或非标准属性。v9 与 v6 算法相同（n-外层循环），仅移除了 OpenMP 和 AVX。N=5000 -O2 下约 12.5s。

### 16.5 v10 循环交换优化

**核心思路**：将外层循环从 n（Garsia-Haglund 递推的行号）交换为 d = n-k（H 行读源）：

```
// v9 (n-外层): 每次 n 迭代读取不同 H[d] 行 → 缓存驱逐
for n = 2..N1:
    for k = 1..n-1:
        d = n-k
        dot(H[d][1..d], qb_diag[k][1..d]) → H[n][k]

// v10 (d-外层): H[d] 行在全部 k 迭代中驻留 L1 缓存
for d = 1..N:
    h_ptr = &H[d][1]  // 固定！所有 k 共享
    for k = 1..N1-d:
        n = k+d
        dot(h_ptr[0..d-1], qb_diag[k][0..d-1]) → H[n][k]
```

**缓存效果**：H\[d] 行（d 个 uint 元素）在 d-内层循环的 N1-d 次迭代中全程驻留 L1。对于大 N，N1-d 可达数千次迭代——每次迭代都省去 L2/L3 访问延迟。
**编译器增强**：`#pragma GCC optimize("O3")` + `#pragma GCC optimize("unroll-loops")` 嵌入源文件，即使判题机用 `-O2` 也启用自动向量化和激进展开。

### 16.6 v11 寄存器压力优化

**核心发现**：32路 ILP 需要 32 个 `ull` 变量 → x86-64 仅有 16 个 XMM 寄存器 → 大量溢出到堆栈。降为 **8路 ILP** 恰好填满 16 寄存器，无溢出。

**实测对比**（N=5000 -O2）：

| ILP          | 时间        | 寄存器使用                       |
| ------------ | --------- | --------------------------- |
| 32路 (v10)    | 9.9s      | 32 ull → 16 XMM + 16 堆栈溢出   |
| 16路 (v11a)   | \~9.2s    | 16 ull → 勉强填充 XMM，少量溢出      |
| **8路 (v11)** | **8.89s** | **8 ull → 完美填充 16 XMM，零溢出** |
| 4×4路 (v11b)  | 10.7s     | 4 \_\_int128 + 4 ull → 过度拆分 |

**v11 全部优化清单**：

1. **d-外层循环** (v10) — H\[d] 行 L1 缓存驻留
2. **预乘 qb\_diag** — `qb[k][r] = powbinom[k]·binom(r+k-1,r)_a`，省 1 次 MOD 每对 (n,k)
3. **8路 ILP** — 消除寄存器溢出（关键 \~0.8s 加速）
4. **`(ull)`** **替代** **`(ll)`** — 消除符号扩展开销
5. **计数循环** — `cnt >= 8` 替代 `p+8 <= limit`
6. **增量 idx 追踪** — `idx_cur += idx_inc` 替代 `n*(n+1)/2 + k`（省乘法）
7. **无手动预取** — 硬件预取器覆盖顺序访问
8. **`#pragma GCC optimize("O3")`** — 内嵌最高优化级

## 十八、v14：NTT 卷积降阶突破 🟢（2025-05-24）

### 18.1 核心突破

**盲点分析文件** [blind-spot-analysis-note0524.md](file:///d:/2024备份和迁移/2024备份和迁移/f复旭文件/h彗星计划相关/其他资料/V6_wida_case/task1/.trae/specs/solve-bracket-dp/blind-spot-analysis-note0524.md) 早在 2026-05-24 就提出了 q-阶乘分离 + NTT 卷积路线，当时被认为是方向 B（已排除的死胡同——因为那个版本试图在 L(i) DP 的 (x,xp) 循环上用 NTT，不适用）。但将其应用到 **Garsia-Haglund 递推的内层求和** 上则完全可行，因为这里的内层是真正的互相关而非对角线操作。

**关键**：v11/v13「空间换时间不可行」论断在 O(N³) 框架内正确——因为 `binom(r+k-1,r)_a` 在 r 和 k 上交叉耦合，三重求和不可消除。**v14 不靠改变循环顺序，而是把求和本身转化成卷积——从代数层面降维。**

### 18.2 数学推导

Garsia-Haglund：

$$H\_{m+k,k} = b^m a^{\binom{k}{2}} \sum\_{r=1}^{m} \binom{r+k-1}{r}_a H_{m,r}$$

q-阶乘展开 q-二项式：

$$\binom{r+k-1}{r}_a = \frac{\text{Fact}\[r+k-1]}{\text{Fact}\[r] \cdot \text{Fact}\[k-1]}, \quad \text{Fact}\[i] = \prod_{j=1}^{i} (a^j - 1)$$

代入：

$$H\_{m+k,k} = \frac{b^m a^{\binom{k}{2}}}{\text{Fact}\[k-1]} \sum\_{r=1}^{m} \underbrace{\frac{H\_{m,r}}{\text{Fact}\[r]}}_{A\_r} \cdot \underbrace{\text{Fact}\[r+k-1]}_{B\_{r+k-1}}$$

令 $A'_i = A_{m-i}$（翻转），则：

$$C\_{m+k-1} = (A' \* B)\[m+k-1] = \sum\_{r=1}^{m} A\_r \cdot B\_{r+k-1}$$

NTT 卷积 → $C\_k$ 对所有 $k=1..N-m$ 在 O(N log N) 完成。

外层 $m=1..N$ → 总 **O(N² log N)**。MOD=998244353（primitive root=3）完美支持 NTT。

### 18.3 实现要点

1. **Fact\[i] 预计算**：`fact[i] = fact[i-1]*(a^{i+1} - 1)`（a=1 时退化为普通阶乘）
2. **B 序列固定**：`B[t] = Fact[t]`，仅需一次 NTT
3. **A 序列按 m 更新**：每个 m 填充 A（翻转 H\[m] / Fact\[r]），做一次 NTT + 逐点乘 + INTT
4. **纯标量 NTT**：Cooley-Tukey 标准迭代实现，无条件分支优化

### 18.9 v14 判题反馈与 bug 修复（2025-05-24 第二轮判题后）

**判题结果**：7/21 正确，14/21 答案错误。

**根因**：q-阶乘 `Fact[n] = ∏(a^i - 1)` 在 a 具有小乘法阶时产生零因子。例：`a = MOD-1 (ord=2)`，`a² ≡ 1` → `Fact[2] = 0`。

**探索过程**：

1. **v11 回退（已废弃）**：最初尝试对坏 a 回退到 O(N³/6) q-二项式递推 + 8路ILP。实测 `a=281849776 (ord=1904), N=5000` 需 10.080s → 但本机 Rtools g++ 编译环境较差，判题机实测 \~9s。
2. **q-Lucas + residue NTT（已放弃）**：试图通过 q-Lucas 定理分解 q-二项式，每个 residue class 做 NTT。数学正确（d=2 全量验证通过），但 d=1904 时 d² 内层循环 336 亿次串行累加（无 ILP 空间），实测 34.7s。
3. **最终方案：双路径架构（2025-05-24 终判）**：

| a 类型          | 占比              | 路径                   | 复杂度         | N=5000 实测   |
| ------------- | --------------- | -------------------- | ----------- | ----------- |
| d > N+2（正常 a） | 99.9993%        | NTT 卷积（原 v14）        | O(N² log N) | **\~7.7s**  |
| d ≤ N+2（稀有 a） | 0.0007%（7168 个） | v11 q-Pascal + 8路ILP | O(N³/6)     | **\~9-10s** |

**为什么不用 q-Lucas**：q-Lucas 的 d² 内层循环对 d=1904 产生 336 亿次必须串行累加的操作，无有效 ILP。而 v11 的 O(N³/6) 点积虽然也是 208 亿次操作，但每次迭代 8 路完全并行+`__int128` 累加器避 modulo，常数远小于 q-Lucas 的 d² 跳跃访问。

**为什么不能 100% 的 a 都 O(N² log N)**：q-阶乘分离本质依赖模逆元，当 a 的阶 ≤ N+2 时 Fact 有零因子，除法不存在。这是群论约束，不是算法设计缺陷。

## 十九、v15：二维自适应 NTT 通解 🟢（2025-05-25）

### 19.1 动因

v14 回退路径 O(N³/6) 在判题机微架构下可能超 10s（GATE-2 风险）。`blind-spot-analysis-note0525.md` 独立分析指出：前期 q-Lucas 方案的「放弃」结论存在盲点——只考虑了商层暴力，未对余数层求和应用 NTT 降维。

### 19.2 核心突破

q-Lucas 分解将坏 a 的 q-binomial 拆为两层独立互相关：

* **余数层**：`H_{m, r₁d+r₀} / Fact[r₀]` 与 `Fact[r₀+k₀-1]` 互相关（NTT 大小 2d）

* **商层**：`I(m, r₁, k₀) / r₁!` 与 `(r₁+k₁)!` 互相关（NTT 大小 2N/d）

两层各一次 NTT，整体 O(N² log N)。自适应阈值（d ≥ 64 用 NTT，否则标量）抹平极端值。

### 19.3 GN-004 审查发现与修正

GN-004 两次审查发现了数学错误：

1. **索引偏移**（第一轮）：原公式 `binom(r₀+k₀, r₀)_a` 应为 `binom(r₀+k₀-1, r₀)_a`（q-Lucas 代入 n=r+k-1）
2. **借位遗漏**（第二轮）：k₀=0, r₀=0 时产生借位 `binom(r₁+k₁-1, r₁)`（非零），不可遗漏

最终公式（`spec.md` §8.2）：

* k₀=0, r₀=0（借位）：`binom(r₁+k₁-1, r₁)`

* k₀=0, r₀>0：0

* k₀≥1, r₀+k₀≤d：`binom(r₁+k₁, r₁) · binom(r₀+k₀-1, r₀)_a`

* k₀≥1, r₀+k₀>d（进位）：0

* d=1 时 a≡1，走对称性 b=1 Garsia O(N²) 特化

### 19.7 v15-b：判题反馈与修复（2025-05-25）🔴→🟡

**v15 判题结果**：19/21 正确，2/21 答案错误（WA）。

**失败用例**：
- N=2000, a=418888083, b=998244352
- N=2000, a=452798380, b=987654321

**根因分析**：检测得 a 的乘法阶 d=2（a ≡ MOD-1，即 998244352），触发四子路径中的"余数标量 + 商层标量"路径。此时 N/d = 1000，远超 NTT 阈值 64，但 d=2 < 64 → 余数层走标量；商层 k₁ 可达 ~1000，但 N/d < 64 时也走标量。问题发生在纯标量路径的内部求和阶段——使用 `ll sum = 0`（signed 64-bit `long long`）作为累加器。

对于 d=2 且 N/d=1000 的用例，余数和商维度均在 NTT 阈值以下时，触发**纯标量路径**。该路径最多约 63 个项（每项 (MOD-1)² ≈ 10^18），累加和 `sum` 远超 `LLONG_MAX`（≈ 9.2×10^18），导致 **signed 64-bit 整数溢出** → 错误答案。

注意：v15 用了 `__int128` 类型变量，但赋给 `ll sum` 累加器时发生截断。两个失败用例的 d 值恰好落入 32 < d < 64 且 N/d < 64 的范围，使得余数和商维度都低于 NTT 阈值，触发纯标量路径。

**修复**：将纯标量路径中的 `ll sum` 和 `ll inner` 全部替换为 `unsigned __int128 sum` 和 `unsigned __int128 inner`。两处修复：
1. 纯标量路径（q-Lucas 节末的 `else` 块）：`ll sum = 0` → `unsigned __int128 sum = 0`
2. 余数-NTT + 商层标量路径：`ll inner = 0` → `unsigned __int128 inner = 0`
