### 阶段 B：2D DP wrapping 修正的开发与证伪

三轮代码迭代尝试用「前导连续 1 个数 c」修正 wrapping：

| 版本 | 修复内容 | 验证结果 |
|------|---------|---------|
| v1 | 纯 naive wrapping | n≤2 正确 |
| v2 | c≥3 dk/db_head 修正 | n≤4 正确，n=5 偏差 828M |
| v3 | dk 偶c r=0/r>0 区分 + db_head 修正为 p(p-1) | n≤4 正确，n=5 仍失败 |

**证伪根因**：2D 状态 `(n, L)` 无法区分同一 (c,r) 分组内不同 x-seq 尾部模式，导致 wrapping 修正因子在聚合时产生交叉污染。**2D wrapping 修正路线已被证伪。**

### 阶段 C：pair_seq 不完全性的大发现

**pair_seq 算法**：扫描序列，`(1,v)→1+v`，`≥2` 保持不变。

**关键发现**：`pair_seq([1]+X(A))` 对「所有元素 ≥2」的 X(A) 成立；对含内部 1 的特定模式**不成立**。

**已知反例**：A=`"()(()())"`（X=[1,2,1]）
- `pair_seq([1,1,2,1]) = [2,2,1]`（bob=4, k=3）
- 真实 `x_seq_and_bob("(()(()()))") = [2,3]`（bob=3, k=2）

**物理原因**：外括号 +1 高度偏差 → `x'ⱼ = max{t: h_A(2l_j-1+t) = t-1}`（标准为 `h=t`），导致高度断点的额外合并。

**wrapping 不可约简性**：变换由 `(x₁, last, k, bob)` 四元组确定（n≤6 验证），无法简化。

## 二、三条已探索路线回顾

### 路线 1：(A)B 分解（放弃）

- 方法：`s = (A) B`，递推 `F[n] = Σ f(k,n)·F[k]·F[n-1-k]`
- 失败原因：bob 步骤边界不跟随 (A)B 括号匹配边界，x₁ 消耗后前缀高度失衡，导致跨步合并（如 `(()(()))` → X=[2,2] 而非 [2,1,1]）

### 路线 2：primitive-concat DP + wrapping 修正（当前 solution.py）

- 方法：`s = P₁·P₂·...·Pₘ`，`G_n(L) = Σ_m a^{m(n-m)}·P_m(L+n-m)·G_{n-m}(L)`
- concat 部分：✅ 完全正确（`X(PQ)=X(P)++X(Q)` 恒成立）
- wrapping 修正：❌ 2D 聚合失败（交叉污染）

### 路线 3：树DP + brute-force wrapping（部分验证）

- 方法：`tree = root + [T₁,...,Tₘ]`，`Dyck = (T₁)(T₂)...(Tₘ)`
- concat 部分：等价于路线 2 ✅
- wrapping 公式（per-word，n≤8 全量验证）：✅
  - x₁≥2：`k((A))=1+kt`，`bob((A))=bobt+(|A|-x₁)`
  - x₁=1：`k((A))=1+k_R(Q)`，`bob((A))=bob_R(Q)+(|A|-1)`
- 树DP 验证：n≤4 14/14 ✅，n=5 41/42（仅 pair_seq 反例失败）

## 五、树DP深化路线图（路线 A 的具体执行方案）

### 阶段 A1：在树上重新推导 bob 的定义 🔑

```
x-seq 的树语义：
- x_j = 从当前消费位置沿「最左链」往下走的深度
- 消费 = 标记该链及其右侧兄弟子树已处理
- bob = Σ(j-1)·x_j

目标：找到 bob 的纯树参数表达式
候选参数：
  - subtree_size[v] = v 的子树大小
  - depth[v] = v 到根的深度（= 括号嵌套层数）
  - left_chain_len[v] = 从 v 一路最左子到底的节点数
  - sibling_index[v] = v 在兄弟中的序位
```

### 阶段 A2：数据驱动分析

- 枚举 n=1..8 全部 Dyck word，对每个树计算：`{叶片深度, 子树大小列表, 最左链长度列表}`
- 寻找 bob 与这些参数的简单算术关系
- 优先尝试：`bob = Σ f(subtree_size[i], depth[i])` 形式

### 阶段 A3：推导树上的 Catalan 递推

- 树 T = root + [T₁,...,Tₘ]
- 若 bob(T) = g(T₁,...,Tₘ) 可分解，则 Catalan 递推直接成立
- 尤其检查 wrapping 在树上是否自然消解

### 阶段 A4：实现并验证

- 用推导出的树公式取代 pair_seq
- 对 n≤8 全量 brute-force 验证
- 若通过，分析时间复杂度并将 O(N³) 优化至 O(N²)

### 若 A 路线陷入死胡同

- 返回到**生成函数猜测**（路线 E 的有限版本）：对特定的 (a,b,n≤10) 枚举值，尝试拟合 Catalan 相关函数
- 考虑**树上的高度断点追踪**（路线 C 的树版本）：不追踪完整 x-seq，只追踪"有多少个 1"和"他们被合并的方式"

## 九、阶段 A1-A4 执行总结（2025-05-21 深入探索）

### 9.1 核心发现

#### 发现 1：concat 公式完全正确 ✅
- `X(PQ) = X(P) ++ X(Q)` 对 n≤8 全量 2055/2055 验证通过
- `bob(PQ) = bob(P) + bob(Q) + k(P)·|Q|` 全量验证通过
- `alice(PQ) = alice(P) + alice(Q) + |P|·|Q|` 已知正确

#### 发现 2：wrapping 公式 X((T)) = pair_seq([1] + X(T)) ✅
- 对 n≤8 全量 2055/2055 验证通过
- 这提供了从 X(T) 计算 X((T)) 的正确方法

#### 发现 3：边界感知 wrapping 算法 ✅
- 基于以下关键洞察：对于已 pair_seq 压缩的 Xw 序列列表
  - [1] 总是与第一个块的首元素合并
  - 仅在块边界处发生额外合并（第 i 块尾=1 且第 i+1 块首元素存在）
  - 块内部不会发生合并（因为每个块已被压缩）
- 算法验证：`wrap_boundary(blocks)` 返回的结果与 `pair_seq([1] + concat(blocks))` 完全一致

#### 发现 4：签名压缩
- 对于 wrapped 形式 Xw(T)，5 参数 `(bobw, kw, n, first_w, last_w)` 可确定 Xw 序列（n≤8，0 次碰撞）
- 不同树的 distinct wrapped 签名按大小增长缓慢：n=8 时 1430 棵 Catalan 树仅有 34 个 distinct 签名

#### 发现 5：原始形式的签名不等于包裹形式 🔴 关键瓶颈
- 对于非原始树：`X(T) = X(P₁) ++ X(P₂) ++ ...`（原始串联），并且 `X((T)) = pair_seq([1] + X(T))`。两个步骤均可行！
- 对于原始树：`X(T) ≠ pair_seq([1] + X(forest))`！公式失败的原因在于：
  - pair_seq 模拟带有高度偏移的 x-seq，但模拟不准确
  - 根节点的 '(' 将高度偏移 +1，这改变了后续 x-seq 步骤中的条件（从 h=t 变为 h=t-1）
  - 这产生了 **内部合并**，而简单的 pair_seq 模拟无法捕捉到
  - 反例：X(forest)=[1,2,1]（对应 Dyck 词 "()(()())"），pair_seq([1,1,2,1])=[2,2,1]，但真实 X(T)=[2,3]

### 9.2 对树形 DP 的影响

原始树代表了**所有**树的根本情况（每个节点都是一个原始形式，包裹其子节点森林）。边界感知算法可以正确地包裹**串联**，但无法包裹从子节点包裹形式构建的**单个原始树**，因为森林 X 与树 X 之间的关系比 pair_seq 更复杂。

这意味着：
- **无法**仅从子节点签名计算 X(T)
- X(T) 取决于通过 x-seq 算法处理的实际 Dyck 词，该算法考虑了高度函数
- 需要完整的 X 序列（或实际的 Dyck 词）来进行原始包裹

### 9.3 待探索的替代方案

1. **批量枚举**：由于每棵大小的不同签名数量增长缓慢（n=8 时为 34），对所有树 ≤8 进行暴力破解，并对更大的 n 插值/外推模式
2. **返回生成函数方法**（路线 D）：接受无法从签名计算包裹，使用关于 X(T) 结构的更完整信息
3. **直接解决原始树**：找到一种闭合形式的组合解释，解释高度偏移如何修改 x-seq，也许通过已知的 Catalan 恒等式

### 9.4 当前结论

树形 DP 路线**部分成功**：它完美地处理了串联和包裹串联。但在包裹单个节点（原始情况）时失败，因为高度偏移。这个问题可能无法通过纯树形参数方法解决，可能需要完全不同的方法。

## 十、竞赛题视角重审 + 路线 G：签名参数空间刺探 🔑

> **背景补充（2025-05-21，用户提醒）**：这是一个 **已知可解的竞赛算法题**，不是未解数学问题。N=5000、模数998244353(NTT-friendly) 都是出题人刻意设定的。预期解使用竞赛中已知的标准技术。

### 10.1 背景重定位对路线的重新评估

在「已知可解」的前提下：

- **路线 C（O(N³) 极限优化）→ 彻底排除**。N=5000 下 4×10¹⁰ 次迭代，即使 C++ 位运算级别也需 ~200-500s，远超 10s 限制。出题人设 N=5000 就是为了卡 O(N³)。

- **路线 A（攻克原始树包裹）→ 方向性存疑**。pair_seq 下原始树 X(T) 的公式需要推导「高度偏移下的跨步合并法则」——这更像数学论文而非竞赛题。出题人大概率不会要求选手在考场上推导一个 x_seq 的偏移合并封闭公式。因此 x_seq/pair_seq 可能一开始就**不是设计 DP 的正确工具**。

- **其他路线回顾**：
  - 路线 B（按 x₁ 分类 + pair_seq）：x₁=1 类有 pair_seq 反例，无法根除
  - 路线 C（高度感知 pair_seq 修正）：需要新增维度，更慢
  - 路线 D（生成函数方程）：L-shift + a^{m(n-m)} 双重耦合未突破
  - 路线 E（猜测通解）：已被用户放弃
  - 路线 F（纯 NTT 卷积）：不可行

- **新方向需求**：在上述路线均有结构性障碍的情况下，需要一种**不依赖 x_seq/pair_seq 框架、能从数据中系统性地回答"DP 需要多少状态"**的方法。

### 10.2 路线 G：签名参数空间刺探 + 线性变换猜想 🆕

**核心思路**：不试图推导 wrapping 的数学公式，而是用 n≤8 的全量数据，**系统性地测试** wrapping 结果的确定性依赖于哪些参数。

#### 为什么这条路与竞赛题背景一致

出题人设计 wrapping 的预期思路可能是：

1. wrapping 确实产生了某些合并效果
2. 但这些合并效果**仅依赖于有限个参数**（如 x₁、x₂、首尾元素、内部 1 的个数等）
3. 因此 DP 状态只需在 `(n, L)` 基础上增加常数个维度
4. 总状态数仍是 O(N²) 或 O(N²·常数) ＝ 可行范围

**需要回答的核心问题**：到底需要几个参数？

#### 三轮刺探方案

**第一轮：参数碰撞穷举**（针对性实验，~10 分钟）

对 n≤8 全量 2055 棵树，收集每个树的 20+ 个候选特征参数，穷举测试哪些参数组合能确定 wrapping 的 `(bobw, kw)`。

候选参数池（从 X 序列中提取）：
```
基础参数：     bob, k, n, first(x₁), last
首尾扩展：    x₁, x₂, x₃, last_2, last_3
统计量：      num_ones, num_ge2, sum_X, sum_sq_X
pair_seq 量：  bob_R, k_R, first_R, last_R
尾模式：      tail_first (X[1:][0]), tail_last, tail_bob, tail_k
链条信息：    leading_ones (连续前导1的个数), total_steps
```

**目标**：找到最小的参数子集 S ⊆ 候选池，使得 `∀ 树A≠B, signature_S(A)=signature_S(B) ⇒ wrap(A)=wrap(B)`（即碰撞数 = 0）。

**第二轮：线性变换猜想**（检验性实验，~15 分钟）

对于每个树计算多种 `(bob, k)` 的线性组合，检查 wrapping 是否会变成恒等式：
```
候选不变量：
  I₁ = bob + k·n
  I₂ = bob - k·(k-1)/2
  I₃ = bob + k·(n - k)
  I₄ = bob + k·(n + 1)
  I₅ = bob - (k-1)·n
  ...以及更多线性组合
```

**目标**：如果存在某个线性变换 `bob' = bob + α·k·n + β·k·(k-1)/2 + γ·k + δ·n` 使得 `bob'((A)) = bob'(A)`（即 wrapping 下不变），那么问题退化为 alice 级别简单——wrapping 只影响 alice 权重（`a^{2n+1}`），bob 不变。

**第三轮：根据前两轮结果决策**（决策分叉）：

| 第一轮结果 | 第二轮结果 | 行动方案 |
|-----------|-----------|---------|

## 十一、路线 G 三轮刺探实验结果 🔑🔑🔑（2025-05-21 执行完毕）

> **实验脚本**：`route_g_probe.py`（对 n≤8 全量 2055 棵树进行系统性数据驱动分析）

### 11.1 核心发现 #1：统一 Wrapping 公式 ✅

**以前错误认知**：x1≥2 时 wrapping 是恒等变换。这是**错误的**！455/1429 棵 x1≥2 的树有非恒等 wrapping。

**正确公式（2055/2055 全量验证通过）**：

```
Xw = pair_seq([1] + X) = [1+x1] + pair_seq(X[1:])

kw   = 1 + tail_R_k
bobw = tail_R_bob + (sum_X - x1)

其中 tail_R = pair_seq(X[1:])，sum_X = Σ X[i]（≠ n_pairs！）
```

**关键洞察**：sum_X ≠ len(s)/2！例如 `()(())` 的 X=[1,2]，sum=3 > n=2。sum_X 是 x-seq 步骤的累加和，必须作为独立参数追踪。

### 11.2 核心发现 #2：Wrapping 仅需 1 个额外参数 🟢

| 参数组合 | 碰撞数 | 分组数 |
|---------|--------|--------|
| (bob, k, n, x1) | 30 | 218 |
| (bob, k, n, x1, **sum_X**) | 30 | 218 |
| (bob, k, n, x1, **tail_R_bob**) | **0** ★★★ | 253 |

**结论**：在基本 DP 状态 (bob, k, n, x1) 上只需增加 **1 个维度** `tail_R_bob`，即可完全确定 wrapping 结果。

### 11.3 核心发现 #3：tail_R 可从 R-签名预测

## 十三、路线收敛根因分析 + 路线 I：x-seq 分解 DP 🔑🔑🔑

> **来源**：在对所有路线（A~H）均受阻于同一瓶颈的反思中，发现了结构性根因。

### 13.1 根因：圆钉砸进了方孔

所有路线（A~H）共享同一个隐藏假设：**DP 分解按照树结构进行**——即父节点包裹子节点森林。

但 bob 函数的**定义性分解**不在树上，而在 **x-seq** 上：
```
x-seq: X = [x₁, x₂, ..., xₖ]
对应 Dyck 词在高度=0 处切分：s = P₁·P₂·...·Pₖ
其中每个 Pⱼ 是 primitive（第一个回到高度0的块）

bob(s) = Σ (j-1)·xⱼ   ← 完全由各块大小和位置决定，无额外计算
```

**两种分解切在不同位置：**

```
s = ((())())
    树分解：1个child（(())()是一整块）
    x-seq分解：X=[3,1] → ((())) | ()
              primitive-1    primitive-2
              切在不同位置！
```

树分解下 alice 完美 ✅，bob 需要 wrapping 修正 ❌。
x-seq 分解下 bob 完美 ✅，alice 需要块间交互处理 ⚠️。

**我们一直在试图把圆钉（x-seq 定义下的 bob）砸进方孔（树分解下的 DP 状态），这就是所有路线卡在同一个 wrapping 步骤的根本原因。**

x-seq 的 pair_seq/Wrapping 只有在 **原始 x-seq 分解**下才是自洽的。一旦我们把它嵌入树分解的框架（子节点 Xw 已被压缩），第二次 wrapping 时信息就不够了。

### 13.2 路线 I：x-seq 分解 DP

**核心思想**：交换分解轴。不在树上做 DP，而在 **x-seq 分解**上做 DP。

**x-seq 分解的 Catalan 对应**：
```
对于任意 x-seq X = [x₁, ..., xₖ]（满足 xⱼ ≥ 1，∑xⱼ = n）：
对应的 Dyck 词 = (Q₁)·(Q₂)·...·(Qₖ)
其中 Qⱼ 是大小为 xⱼ-1 的任意 Dyck 词
```

**在这种分解下**：

| 操作 | 公式 | 说明 |
|------|------|------|
| **bob** | `bob = Σⱼ (j-1)·xⱼ` | 直接由 x-seq 决定，完全免费！ |
| **alice** | `alice = Σⱼ alice((Qⱼ)) + Σ_{i<j} xᵢ·xⱼ` | 块间交叉项 + 各块内部 |
| **块内 alice** | `alice((Qⱼ)) = alice(Qⱼ) + 2xⱼ - 1` | wrapping 只增加 alice（bob 不变！） |
| **K** | `k = Σⱼ 1 = 块数` | 步数 = x-seq 长度 |

**为什么这能绕过瓶颈**：

x-seq 分解下，wrapping「包一对括号」**不改变 bob**！回顾：
```
D → (D)：在 x-seq 分解下，D 的 x-seq 不变，只是外加了一层括号
bob(D) = Σ (j-1)·xⱼ → bob((D)) = 同样的值
alice((D)) = alice(D) + 2|D| + 1
```

这意味着：**在 x-seq 分解框架下，bob 是分块可加的，不需要任何 wrapping 修正！**

**DP 状态**：`F[n]` = 所有大小为 n 的 Dyck 词的贡献（仍然需要生成函数形式）

**未解决的挑战**：
- alice 的块间交叉项 `xᵢ·xⱼ` 在 DP 中如何高效处理？
- 这引出一个类似的标准 DP 问题：给定 x₁..xₖ 满足 ∑xⱼ=n，求所有分拆下 `a^{Σ_{i<j}xᵢxⱼ}·b^{Σ(j-1)xⱼ}` 的和

## 十五、全局反思与路线 D 重启 🔑（2025-05-21→2025-05-22）

> **背景**：路线 I 证伪后，所有基于 x-seq/pair_seq 分解的 DP 路线全部关闭。重新审视题目设定、已知数据和竞赛题背景，识别新的方向性线索。

### 15.1 问题的几何化重述

Dyck 词等价于**格点路径**：从 (0,0) 到 (2n,0)，每步 (1,±1)，不穿过 x 轴。

**alice(s) 的路径意义**：
```
alice = #{i<j : s_i='(', s_j=')'}
      = Σ_{上升步i} (i右侧下降步的个数)
```

这与路径的**累积面积/水位**相关——下降步越早出现，alice 越小。

**bob(s) 的路径意义**：
```
xⱼ = 从高度hⱼ回到同一高度的水平excursion width
bob = Σ (j-1)·xⱼ
```

bob 衡量的是路径中**各次返回高度0的excursion宽度**的加权和。

### 15.2 核心线索汇总（来自题目设定、过程和数据的暗示）

#### 线索 1：NTT-friendly 模数（998244353 = 119 × 2²³ + 1）

**含义**：出题人期望解中使用**卷积/多项式运算**。不是巧合——这个模数就是为 NTT 选的。

#### 线索 2：输出全部 F₁..Fₙ

**含义**：不是单点求值，而是**递推式提取系数**。标准模式是：生成函数满足方程 → 牛顿迭代 O(N log N) 提取系数。

#### 线索 3：alice 和 bob 的"交换对称性"

| 视角 | alice | bob |
|------|-------|-----|
| 树分解（父→子森林） | ✅ 完美递推 | ❌ wrapping 修正 |
| x-seq 分解（primitive 块队列） | ❌ 交叉项 | ✅ 定义性完美 |

**含义**：alice 和 bob 可能是某个**更基础对象**（Dyck 路径本身）的两种不同投影——投影到树得到 alice，投影到 x-seq 得到 bob。两者不能同时简化不是 bug，是**结构性对偶**。

#### 线索 4：wrapping 取决于完整高度函数（数据证实）

**含义**：wrapping 是**全局操作**，不能从局部参数恢复。在"分解-组装"范式下无法处理。标准应对方式：**不分解，建立生成函数方程**。

#### 线索 5：题目是已知可解的竞赛题

**含义**：解法使用已知的标准竞赛技术组合（非研究级突破）。生成函数方程 + 牛顿迭代是候选之一。

### 15.3 路线 G 重启可行性分析

路线 G 的方法论正确——用数据系统测试参数依赖性。错误在于用 pair_seq 而非真实 wrapping 作为地面真相。

**重启 G 的关键变化**：参数池必须扩展到 x-seq 之外：

| 参数来源 | 候选 | 维度 |
|---------|------|------|
| 高度函数统计 | h_max（最大高度）、h_sum（高度积分/路径面积）、h_moments | O(1) |
| 树的直接参数 | 子树大小列表、深度列表、叶子数 | O(n) |
| 面积相关 | 路径下方面积、首次返回各高度位置 | O(n) |

**核心问题**：区分同 x-seq 分组需要多少额外参数？如果是 O(1)（如 h_max + 面积），DP 仍然可行。

### 15.4 路线 F 不可行性确认（数学障碍）

递推公式 `G_n(L) = Σ_m a^{m(n-m)}·P_m(L+n-m)·G_{n-m}(L)` 有三个不可分离的耦合：

| 耦合 | 位置 | 性质 |
|------|------|------|
| `a^{m(n-m)}` | m 和 n 耦合 | 加权卷积，非标准 |
| `P_m(L+n-m)` | L-shift | 无法预计算 |
| 外层 L 循环 | 每次 L 独立卷积 | O(N)·O(N log N) = 过高 |

**这本质不是卷积**——不是技巧问题，是数学障碍。分治 NTT（CDQ）处理在线卷积，但不处理加权在线卷积。**路线 F 已永久关闭。**

### 15.5 🟢 D（生成函数方程）+ J（bob 等效表达，已闭合）

所有 A~I 路线失败的根本原因已定位：wrapping 是全局操作，不可从任何低维参数恢复。"分解-组装"范式不适用。在此认识下，两条性质不同的路线并行推进：

#### 路线 D：生成函数方程（理论推导）

最早定义于 §三，2025-05-21 用户决策暂缓，但从未被证伪。J-2（生成函数方程探索）与 D 是同一方法——已合并回 D。

**方法**：对 `F(x,y) = Σ a^{alice} b^{bob} x^{|s|} y^{k(s)}` 建立函数方程，利用 Dyck 路径首次返回高度 0 处的分解（= primitive 分解），通过 Lagrange 反演或核方法（kernel method）求解，Newton 迭代 O(N log N) 提取系数。

**线索支撑**：NTT 模数（998244353=119×2²³+1）+ 输出全部 F₁..Fₙ → 强烈暗示此方向。生成函数方程天然处理全局操作——无需分解，直接建方程。

**方法**：用 n≤8 全量数据，计算每个 Dyck 词的几何量（路径面积/高度积分、树参数 Σ f(size, depth) 等），穷举测试 bob(s) 是否等于某个简单表达式。

**如果成功**：bob 变成像 alice 一样可在树分解下递推的量——wrapping 问题消失，标准 Catalan DP 直接可用。

#### 15.6.3 根因确证

Wrapping 是**全局操作**——依赖于完整的 Dyck 词高度函数 `h_A(t)` 在整个前缀 `[0..2|A|]` 上的取值。任何有限维度的参数集都会丢失高度函数的细节信息，而这些细节在某些前缀处决定了 wrapping 的合并行为。

**这不是"参数不够多"的问题，而是"信息维度不匹配"的问题：**
- 输入：2|A| 维高度函数
- 输出：wrapping 结果 (bobw, kw)
- 中间映射：不可逆的高度维度压缩
- 任何 O(1) 参数集能携带的信息量远小于完整高度函数

### 15.8 路线 J 执行结果：bob 无几何等效表达 🔴（2025-05-22）

> **实验脚本**：`route_j_probe.py`（对 n≤8 全量 2055 词进行系统性穷举测试）

#### 15.8.1 实验设计

计算每个 Dyck 词 s 的 25+ 几何/树参数，穷举测试 bob(s) 是否可用这些参数的简单表达式表示。

**参数池**：

| 来源 | 参数 | 说明 |
|------|------|------|
| 高度/路径 | h_max, h_sum, returns0 | 路径最大高度、面积、回零次数 |
| 树统计 | sum_depths, sum_subtree_sizes, leaf_count, max_depth | 节点深度和、子树大小和、叶数、最大深度 |
| 兄弟/遍历 | sum_sibling_indices, sum_parents_before, sum_chain_indices | 兄弟序位和、祖先左兄弟和、x-seq链索引和 |
| 子树 | max_subtree_size, sum_sq_subtree_sizes, sum_depth_times_sz | 最大子树大小、子树大小平方和、深度×大小 |
| x-seq | k, x1, internal_ones | 步数、首元素、内部1的个数 |
| 组合 | n_sq (=n²), n_choose_2 (=n(n-1)/2), alice | n², C(n,2), alice值 |

**测试类型**：
1. 单参数直接相等：bob == param ?
2. 双参数仿射组合：bob == α·p₁ + β·p₂ (+ const) ?
3. 三参数线性组合：bob == α·p₁ + β·p₂ + γ·p₃ ?
4. 非线性候选：n²-alice, n(n-1)/2-param, h_sum/2, h_sum-sum_depths 等 20+ 表达式
5. 相关性排名：Pearson 相关系数

#### 15.8.2 关键反例

同一组 (k, returns0, leaf_count) 可对应不同 bob 值：

```
n=3: k=2, returns0=1, leaf_count=2
  → bob=1 (X=[2,1], alice=7)
  → bob=2 (X=[1,2], alice=7)

n=4: k=2, returns0=0, leaf_count=2
  → bob=1 (X=[3,1], alice=15)
  → bob=2 (X=[2,2], alice=14)
```

**根因**：bob = Σ(j-1)·xⱼ —— xⱼ 的**次序**决定加权系数。X=[2,1] 和 X=[1,2] 有完全相同的参数多集（同为 {1,2}），但排序不同导致 bob 不同（1 vs 2）。任何基于聚合/求和/平均的参数都丢失了排序信息。

| 4 | 14 | 7 | 2.0:1 |

| 6 | 132 | 16 | 8.2:1 |
| 7 | 429 | 22 | 19.5:1 |
| 8 | 1430 | 29 | 49.3:1 |

bob 值的增长远慢于 Catalan 数——但不足以让 DP 状态空间可管理，因为 wrapping 还需要额外信息。

| 排名 | 参数 | 相关系数 | 解读 |
|------|------|---------|------|
| 1 | k | 0.893 | x-seq 步数——自然相关（bob 定义涉及 k） |

### 15.9 路线 K：c-based wrapping 递推 🔑🟢（2025-05-23 更新）

> **2025-05-23 重大突破**：原先的 Orbit Fibonacci 结论基于旧版字符级 x_seq，已修正。c-based wrapping 公式 255/255 验证通过。

#### 15.9.1 L(i) 序列表示 — 替代所有旧框架

定义 `P(i)` = 第 i 个右括号位置（1-based），`L(i) = P(i) - i`。

三重公式（n≤5 全量验证）：
- `alice = Σ L(i)` — 纯线性，零分解
- `l₀=0, lⱼ₊₁ = L(lⱼ+1)` — x-seq 消费点的一阶递推
- `bob = Σ j·(lⱼ₊₁ - lⱼ)` — 由 L 直接算

#### 15.9.2 x_seq concat — 确认无边界效应

x_seq 元素以括号对计量。n≤7 全量验证：
```
X((A)B) = X((A)) ++ X(B)    ✓
```
wrapping 复杂度被完全隔离到单层包裹 H 上。

#### 15.9.3 ✅ C-based Wrapping 公式 — 核心突破

任何 Dyck word A 的 wrapping 结果 (bw,kw) 由两个参数确定：
1. **c** = X(A) 中前导连续 1 的个数（= A 的前导 `"()"` primitive 块数）
2. **(bw_rest, kw_rest)** = 去除前导 1 后 X[c:] 的 wrapping 参数

**公式**（n=1..8 全量 255/255 验证通过）：
```
c = 0 (x₁ ≥ 2):   非恒等 — 需通过 rest wrapping 递推
c = 2p (偶数):     bw = p(k-p) + bw_rest,        kw = p + kw_rest
c = 2p+1 (奇数):   bw = (p+1)(k-p-1) + bw_rest,  kw = p+1 + kw_rest
c = k (全 1):      退化情况：t=k+1→偶数:bw=u(u-1),kw=u; 奇数:bw=u²,kw=u+1
```

**Alice 递推**（同样验证通过）：
```
alice(A) = alice(B) + c·(c+1)/2 + c·(k-c)
```

## 十六、路线 L：L(i)序列生成式DP — 范式转换 🔑🟢（2025-05-23 确立）

> **来源**：`blind-spot-analysis-note0523.md` 独立分析。经 n=1..7 全量数据交叉验证，所有核心数学论断成立。

### 16.1 范式转换：终结 A~K 的结构性矛盾

**所有路线 A~K 的公共死因**：

```
bob定义在x-seq轴上：bob = Σ(j-1)·xⱼ
                         = 各primitive块的(位置序)×(块大小)

所有路线A~K在Dyck词上做分解：树/primitive/(A)B/wrapping/orbit
                            这些分解的边界 ≠ x-seq块的边界
```

以 `(()())` 为例：
- 树分解：1个子树（整个结构是一块）
- primitive分解：1个primitive块（只在末尾回零1次）
- x-seq分解：[2,3]或[2,2,1]（取决于wrapping），2-3个块
- **三种分解切在完全不同的位置！**

路线A~K试图在Dyck词分解上处理bob（x-seq轴线上的量），必然产生wrapping修正——而wrapping取决于完整高度函数，不可局部压缩。

### 16.2 L(i)序列：完备且无分解的表示

```
P(i) = 第i个右括号在字符串中的位置（1-based）
L(i) = P(i) - i
```

**1:1对应**：每个Dyck词对应一个L(i)序列，反之亦然。

**约束**（充要条件）：
- 单调不减：L(i+1) ≥ L(i)
- 边界：L(i) ∈ [i, n]
- 终点：L(n) = n

**两个核心公式**（n=1..7全量验证）：
- `alice = Σ_{i=1..n} L(i)` — 无需任何分解
- `bob = Σ_{j=1}^{k-1} (n - lⱼ)`，其中 l₀=0, lⱼ₊₁=L(lⱼ+1) — 每跳贡献独立

### 16.3 跳跃链的马尔可夫性

**跳跃点序列**：
```
l₀ = 0
l₁ = L(1)
l₂ = L(l₁+1)
l₃ = L(l₂+1)
...
lₖ = n
```

**一阶性**：下一个跳跃lⱼ₊₁仅取决于当前跳跃点前一位的L值=L(lⱼ+1)，与历史无关。

**bob贡献**：第j次跳跃贡献n-lⱼ（完全不依赖j！）。这意味着DP状态无需跟踪k（步数）。

### 16.4 DP设计

**状态**：dp[x][y] — 当前跳跃点为x，且L(x)=y。初始：dp[0][0]=1。

**转移**：
1. 从(x,y)出发，下一跳跃触发点=y+1
2. 中间段x+1..y：填单调非降L值，仅贡献alice（a^{ΣL}）
3. L(y+1)=y'成为下一跳跃值
4. 转移：dp[y+1][y'] += dp[x][y] × a^{alice_mid} × b^{n-x}

**中间段计数**：
- 位置x+1到y，共(y-x)个位置
- L(x+1..y)单调非降，L(x+1)≥y，L(y)≤y'
- 每个L(i)贡献a^{L(i)}到alice

### 16.5 复杂度与优化路径

| 实现 | 时间 | 说明 |
|------|------|------|
| 直接实现 | O(N³) | dp[x][y] 对每个y'∈[y+1,n]转移 |
| 前缀和优化 | O(N²) | 将Σ_{y'} dp[y+1][y']×C(x,y,y')转为前缀和 |
| 空间 | O(N²) | N=5000时约200MB，在600MB内 |

**优化思路**：dp[y+1][y'] = Σ_{x≤y} dp[x][y] × C(x,y,y') × b^{n-x}
- C(x,y,y')仅依赖(x,y,y')，可预处理
- 固定y和y'时，对x的求和可前缀和加速

### 16.6 为什么这应该可行

1. **表示完备**：L(i)↔Dyck词是1:1，无信息丢失
2. **无wrapping**：L(i)框架下没有"包裹"操作——每个L(i)值独立存在
3. **2D状态**：仅需(x,y)，不含k、不含wrapping参数
4. **组合可计算**：中间段是非降序列计数，是经典组合问题
5. **出题信号一致**：2D DP + 组合预处理 + O(N²)，符合竞赛题预期复杂度

### 16.7 方向收敛：q-Dyck wrapping 生成函数（2025-05-23 终判）

路线 L 的 DP 框架在数学上完全正确（n≤14 验证），但 H[k][L] 的计算依赖于 wrapping 的全体加权和。进一步分析揭示：wrapping 是原问题的**最小不可约单元**——所有13条路线均在此受阻，原因是 wrapping 天然等价于完整高度函数（信息维度 O(2^k)），不可由低维参数压缩。

问题被等价归约为：

$$C[k][kw] = \sum_{A: |A|=k,\ k((A))=kw} a^{\text{alice}(A)} \cdot b^{\text{bob}((A))}$$

此即「q-Dyck 的 wrapping 生成函数」——按子树大小 k 和 wrapped 步数 kw 双变元分组的加权 Catalan 和。

生成函数方程已建立：
$$F(x) = 1 + \sum_{k \ge 0,\ kw} C[k][kw] \cdot a^{2k+3} \cdot x^{k+2} \cdot F(a^{k+2} \cdot b^{kw} \cdot x)$$

下一步：为 C[k][kw] 建立递推关系或独立的 GF 方程。候选路径包括：从 k≤14 数据反推递推模式，或建立 C(y,z) 的双变元 GF 方程。

### 十七、q-二项式分析：blind-spot-analysis-note0523v2.md 的独立验证（2025-05-23 补录）

#### 17.1 笔记主张

`blind-spot-analysis-note0523v2.md` 提出：
1. 中止 C[k][kw] 路线，转回 L(i) 序列 DP
2. H[k][L] 可用 q-二项式系数替代
3. DP 转移可重构为 NTT 友好的卷积形式：`dp[y][z] = b^{n-y} Σ dp[x][y] · a^{y(y-x)} · binom(z-x-1, z-y)_a`
4. 总复杂度可降至 O(N² log N)

#### 17.2 独立验证结果（verify_qbinomial_claim.py）

**✅ 正确部分**：
- H[d][u] = binom(d+u, d)_a。路线 L 已通过 `precompute_H_and_pow` 在 O(N²) 内计算 q-二项式系数。
- 对 N=10, a=2 全量 121 项验证一致。

**❌ 不成立部分**：
- **状态简化错误**：dp[y][z]（仅追踪跳跃点）丢失了 L(jump_point)。L(x') 不是 x' 的函数——x' 是跳跃点值而非字符串位置，无法从 x' 推导 L(x')。
- **公式不完整**：`binom(z-x-1, z-y)_a` 不依赖 y'=L(x')。而已验证的 DP 中，中间段上界正是 y'。
- **NTT 不适用**：H[d-2][yp-xp] 是对角线运算（仅依赖 yp-xp 差值），不是交叉相关。固定 d 和 xp 时，所有 yp 的转移通过逐点乘法在 O(N) 内完成——不需要卷积。

#### 17.3 有效部分

q-二项式身份确认本身有用：
- H 的 q-二项式生成函数闭式已知：Σ_{d≥0} binom(d+u, d)_a · t^d = 1/∏_{i=0}^u (1 - a^i t)
- 此闭式或许可在生成函数层面消去 L 维度
- H 的递推 `H[d][u] = H[d][u-1] + a^u · H[d-1][u]` 已是最优 O(N²) 计算方式

#### 17.4 对路线的影响

- 路线 L 的 DP 框架（dp[x][y] 状态 + H 中间段系数）数学正确且已优化
- O(N²) 降阶不在于加速 H 计算（已是 O(N²)），而在于减少外层循环的 (x, xp) 组合数或 yp 维度
- 当前未闭合的降阶路径：
  1. yp 维度的前缀和（固定 x, xp 时 O(1) 转移，而非 O(N)）
  2. G_n(y) = y · Σ_s A[s] · a^{s(n-s)} · G_{n-s}(b^s y) 的变量消去
  3. q-二项式的生成函数恒等式用于闭合形式推导

#### 18.5 本质障碍

`dp[xp][*]` 的转移需来自**每个前驱 x** 的独立贡献：

```
dp[xp][xp+u] += Σ_{x=0}^{xp-1} C(x, xp) · G_{xp-x}[u]
```

C(x, xp) = S[x][xp] · b^{n-x} · a^{(xp-x)·xp} 对每个 x 不同，且 S[x][xp] 由 DP 在线产生（不可预计算）。这本质上是 (x, xp) 两两组合的遍历——除非存在跨 x 的求和恒等式或生成函数变换，否则 O(N²) 不可达。
