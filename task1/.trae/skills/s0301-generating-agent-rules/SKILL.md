# s0301 AGENTS 规则模板生成

## 作用
- 把 `s0201` 的契约边界与 `s0203` 的模块边界压缩为可安装、可审查的 AGENTS 规则模板宏。
- 为项目根级与模块级规则提供统一骨架，避免后续协同执行依赖聊天记忆或临时口头约束。
- 把"规则是否能真实约束执行边界"作为主问题，而不是把 AGENTS 写成泛泛项目说明。

## Trigger
- 用户要求生成、补齐或重构 `AGENTS.md` 类规则文件。
- 契约与模块拆分已形成，需要把规则入口从"隐性约束"落成"显性模板"。
- 当前任务即将进入多执行者协作、规则注入或长期维护，但还没有统一的规则载体承接上下文保留。

## Preconditions
- 已存在可读取的契约边界输入，如 `s0201` 的三层契约结果或等价冻结说明。
- 已存在模块拆分结果，如 `s0203` 的模块职责、依赖范围与回退锚点。
- 当前任务确实需要生成规则模板宏；若只是讨论规则理念或未形成契约/模块边界，不启动本 Skill。
- 若无法确认项目根级规则权限、模块粒度或契约来源，先标记 `未闭合` 或 `当前不可判定`，不伪造正式模板。
- 若当前需求只是把既有规则或模板真正写入文件，而不是先生成规则模板宏，优先转入 `s0401` 或正式落位阶段，不在本 Skill 内越权落盘。

## TRAE 特化命令
- 若规则作用范围、全局/模块层级归属、上下文保留优先级仍有歧义，必须先调用 `AskUserQuestion` 做结构化澄清，不能直接编一版"看起来合理"的 AGENTS 模板。
- 若本 Skill 已产出可供后续写入的模板宏或明确阻断说明，应调用 `TodoWrite` 同步 `已闭合 / 未闭合 / 当前不可判定` 的真实状态。
- 本 Skill 的模板抽取与对齐只能依赖本地契约、模块拆分、既有规则与当前 spec 资产完成；检索仅使用 `Read` / `Grep` / `Glob` / `SearchCodebase`。

## Context
- 本 Skill 产出的是 AGENTS 模板宏与规则绑定骨架，不替代真实业务文档，也不替代契约定义本身。
- 模板必须同时覆盖根级规则与模块级规则的最小必需结构，且显式区分谁可修改、谁只能生成。
- 模板必须承接"规则优先级最高""上下文永久保留""契约优先""模块边界隔离"四类硬约束。
- 模板不得要求修改 `public/` 下内容，不得制造跨模块直接导入的灰色地带。
- 规则写法必须让后续主执行者在加载后可独立完成首步动作：先识别规则层级、再回填约束来源、再决定写入位置。
- 本 Skill 统一以 `AGENTS.md` 为根级与模块级规则载体命名，不再接受 `AGENT.md` 分支写法。

## Inputs
- 三层契约结果或契约摘要。
- 模块拆分表、模块职责边界、依赖关系与可修改范围。
- 既有全局规则、模块规则样式与上下文保留规则。
- 可选的项目专属日志规范、错误码规范、测试闸门要求。

## Outputs
- 根级 `AGENTS.md` 模板宏。
- 模块级 `AGENTS.md` 模板宏。
- 规则字段绑定表：每个模板段落对应哪些契约/模块输入。
- 三值状态、未闭合项、写入入口与后续接续点。

## Action Flow
1. 读取契约结果与模块拆分结果，锁定规则需要承接的真实边界，而不是先写模板空话。
2. 提炼全局层必须常驻的规则段：优先级声明、上下文保留声明、契约优先、公共禁改、跨模块依赖边界。
3. 提炼模块层必须独占的规则段：可改路径、依赖 Mock / 契约入口、测试要求、失败回退与升级入口。
4. 为每个段落建立"规则字段 ← 契约/模块输入"的绑定表，避免模板与真相源脱钩。
5. 若发现层级归属冲突、模块边界不清或契约仍未冻结，调用 `AskUserQuestion` 对齐，或明确保持阻断状态。
6. 输出根级模板宏、模块级模板宏、绑定关系、三值状态与下一步写入建议。
7. 调用 `TodoWrite` 回填当前是否已具备进入规则注入或正式写文件阶段的条件。

## hc_sc_ec_mapping
- 主映射：`HC-2` `HC-3` `EC-3`
- 次映射：`SC-2` `SC-3` `EC-6` `EC-7`
- 说明：
  - `HC-2`：把契约与模块边界分解为可被规则直接承接的模板段落。
  - `HC-3`：不仅输出模板，还要说明当前规则冻结到哪里、哪些仍未闭合。
  - `EC-3`：模板必须可被 GN-004 或后续写入动作独立核对，而不是只靠执行者自述。
  - `SC-2`：明确根级与模块级的顺序依赖与层级边界。
  - `SC-3`：边界不清时不能把模板伪装为已可写入。
  - `EC-6/EC-7`：发现规则归属冲突或权限不明时必须解释并请示。

## State Protocol
- `已闭合`：根级与模块级模板宏均已形成，绑定关系清楚，可进入正式写入或注入阶段。
- `未闭合`：模板骨架已形成，但仍有层级归属、模块范围或规则来源冲突，不能直接下推。
- `当前不可判定`：当前缺少足够的契约或模块输入，无法可靠判断模板是否成立。

## Closure Signals
- 存在根级模板宏与模块级模板宏两套结果，而不是只给一段泛规则说明。
- 模板显式包含四个强制部分：优先级声明、上下文保留声明、AC 范式通用约束、层级专属约束。
- AC 范式通用约束至少覆盖四条：禁止修改 `public/`、跨模块仅依赖 `public/` 契约、所有数据读写必须通过公共契约校验、所有对外接口必须严格匹配契约。
- 已给出规则字段绑定表，能回溯每段模板对应的契约/模块输入。
- 明确给出三值状态、未闭合项与后续写入入口。
- 若状态不是 `已闭合`，必须说明为什么当前不能进入正式写入。

## hc3_handoff
- 做到哪里：已把契约与模块边界压缩为 AGENTS 模板宏，或说明当前为什么无法可靠压缩。
- 为什么做到这里：因为后续协同执行需要一个在上下文压缩中仍会被保留的显性规则入口。
- 哪些已闭合：已冻结的全局规则骨架、模块规则骨架、模板字段绑定关系。
- 哪些未闭合：未裁定的模块范围、未确认的项目专属规则、仍漂移的契约边界。
- 下一步从哪接：
  - 若 `已闭合`，进入正式写文件或规则注入动作
  - 若非 `已闭合`，先补边界裁决或回退到契约/模块拆分节点

## ec7_trigger_gate
- 无法确认全局规则与模块规则的职责归属。
- 契约边界与模块边界彼此冲突，导致模板段落无法稳定绑定。
- 用户要求直接写入规则文件，但当前模板还不能作为可靠真相源。

## ec6_minimum_explanation
- 当前为什么可以或不可以生成可写入的 AGENTS 模板。
- 哪些规则段已经能稳定落位，哪些仍缺契约或模块边界支撑。
- 已完成了哪些绑定核对与层级分离动作。
- 建议下一步由谁补齐什么，才能继续进入写入或注入阶段。

## gn004_touchpoints
- 三件套阶段：检查该 Skill 是否被定义为"规则模板生成器"，而非再次发明总控编排器。
- 关键审查点阶段：检查模板层级、绑定关系、三值状态与 HC-3 交接字段是否齐备。
- 执行阶段：检查执行者是否真的依据模板宏生成规则，而不是回到临时聊天约束。
- 交付前审查：检查最终规则文件是否仍可回溯到本模板与契约/模块输入。
- 所有 gn004_touchpoints 段落中的审查动作均为主线程拉起 GN-004，不得暗示 subagent 可自行拉取。

## installability_validation
- `frontmatter` 存在且字段可读。
- `description` 能命中"AGENTS / 规则模板 / 规则固化 / 上下文保留"类触发语义。
- 首步动作明确为"读取契约与模块边界并拆成根级/模块级模板段落"，主执行者可独立执行。
- 若当前环境无法验证触发命中，则状态记为 `当前不可判定`。

## validation_record_format
- `验证对象`：`s0301-generating-agent-rules`
- `验证时间`：记录实际执行时间戳。
- `frontmatter 合规`：`通过 / 失败 / 当前不可判定`
- `description 触发命中`：记录命中的触发语义、未命中原因或 `当前不可判定` 理由。
- `首步独立执行`：记录是否能直接从"读取契约与模块边界并拆分模板段落"启动，以及阻断点。
- `模板层级覆盖`：记录根级模板、模块级模板、绑定表、优先级声明、上下文保留声明是否齐备。
- `AGENTS 强制骨架核对`：记录四个强制部分与 AC 通用约束四条是否全部写入模板宏，而不是留给后续执行者自行补齐。
- `总体结论`：只能填写 `已闭合 / 未闭合 / 当前不可判定` 之一。
- `证据与备注`：记录引用文件、冲突项、未闭合项与后续接续入口。

## fallback_and_escalation
- 若契约或模块拆分尚未冻结，先输出模板缺口与阻断说明，不强行生成正式规则宏。
- 若全局/模块层级冲突，保留冲突与请示入口，不把伪模板交给后续写入动作。
- 若模板缺少优先级声明、上下文保留声明或层级专属边界任一关键段，直接保持 `未闭合`。
- 若模板缺少四个强制部分中的任一项，或 AC 通用约束四条没有全部落位，直接保持 `未闭合`。
- 若当前动作已经从"模板生成"切换为"真实写文件"，直接退避到 `s0401` 或正式落位阶段，不在本 Skill 内混做写盘。
- `EC-4` 间接补偿：由 GN-004、AskUserQuestion 与主 agent 复核防止模板偏航或越权扩写。
- `EC-5` 间接补偿：若当前能力不足以裁决规则归属或项目专属约束，应升级请示，而不是自造规则真相源。
