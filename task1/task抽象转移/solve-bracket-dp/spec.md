### 4.4 v14：NTT 卷积降阶（2025-05-24 突破）🟢

v14 通过 **q-阶乘变量分离** 从代数层面化解了三重求和：

$$H\_{m+k,k} = b^m a^{\binom{k}{2}} \sum\_{r=1}^m \binom{r+k-1}{r}_a H_{m,r}$$

$$\downarrow \binom{r+k-1}{r}\_a = \frac{\text{Fact}\[r+k-1]}{\text{Fact}\[r] \cdot \text{Fact}\[k-1]}$$

$$H\_{m+k,k} = \frac{b^m a^{\binom{k}{2}}}{\text{Fact}\[k-1]} \sum\_{r=1}^m \underbrace{\frac{H\_{m,r}}{\text{Fact}\[r]}}_{A\_r} \cdot \underbrace{\text{Fact}\[r+k-1]}_{B\_{r+k-1}}$$

→ $A^{\text{rev}} \* B$ 通过 NTT 在 O(N log N) 完成，外层 $m=1..N$ → **O(N² log N)**。

**性能基准**（g++ -O2, Ryzen 7840HS 3.8GHz）：

| ---- | ------------- | ----------------- | -------- |
| 500  | 3.5s          | **0.06s**         | **57×**  |
| 1000 | \~28s         | **0.18s**         | **156×** |

### 六-B、v14 第二轮判题反馈与 bug 修复（2025-05-24）

**判题结果**：7/21 正确 ✅，14/21 答案错误 ❌。

**根因**：q-阶乘分离 `Fact[n] = ∏(a^i - 1)` 在 a 具有小乘法阶时产生零因子 → `0/0` 导致错误。特别是 `a = MOD-1 (ord=2)` 时 `a^2 ≡ 1` → `Fact[2] = 0`。

**修复**：双算法架构。预检测 `(a^i - 1) ≡ 0`，若发现则回退到 q-二项式递推（q-Pascal recurrence，无除法）+ v11 风格 8路 ILP。否则保持 NTT 卷积。

**验证**：128 次 Python 镜像测试全量通过（50 NTT + 40 回退 + 48 BF），包括 N=100 NTT 和 N=30 回退。

### 8.2 q-Lucas 分解（修正公式）

设 a 的乘法阶为 d。对 binom(r+k-1, r)\_a，令 r = r₁d + r₀, k = k₁d + k₀（0 ≤ r₀, k₀ < d）。

则 n = r+k-1 = (r₁+k₁)d + (r₀+k₀-1)。分情况：

| 条件                | n₁      | n₀        | binom                                        | 说明                   |
| ----------------- | ------- | --------- | -------------------------------------------- | -------------------- |
| k₀ = 0, r₀ = 0    | r₁+k₁-1 | d-1       | **binom(r₁+k₁-1, r₁)**                       | 借位，n₀ = d-1 ≥ 0 = r₀ |
| k₀ = 0, r₀ > 0    | r₁+k₁-1 | r₀-1      | **0**                                        | n₀ < r₀              |
| k₀ ≥ 1, r₀+k₀ ≤ d | r₁+k₁   | r₀+k₀-1   | **binom(r₁+k₁, r₁) · binom(r₀+k₀-1, r₀)\_a** | n₀ ≥ r₀              |
| k₀ ≥ 1, r₀+k₀ > d | r₁+k₁+1 | r₀+k₀-1-d | **0**                                        | 进位，n₀ < r₀           |

> **注意**：`k₀ = 0, r₀ = 0` 的借位情形在 d 较小时频繁出现（如 r,k 同为 d 的倍数），不可遗漏。binomial 因子替换为 binom(r₁+k₁-**1**, r₁)（非 binom(r₁+k₁, r₁)）。

> **d = 1 退化**：当 a ≡ 1 (mod MOD) 时 d=1，可利用 C\_n(a,b)=C\_n(b,a) 对称性，走 b=1 Garsia O(N²) 特化路径。此边界不在二维 NTT 框架内处理。

### 8.3 二维互相关结构

**余数层**（固定 m, r₁）：

* **k₀ ≥ 1**（常规非零项）：

$$I(m, r₁, k₀) = \frac{1}{\text{Fact}\[k₀-1]} \sum\_{r₀=0}^{d-k₀} \frac{H\_{m, r₁d+r₀}}{\text{Fact}\[r₀]} \cdot \text{Fact}\[r₀+k₀-1]$$

→ NTT 卷积：A\[t] = H\_{m, r₁d+t} / Fact\[t]，翻转后与 B\[t] = Fact\[t] 卷积

* **k₀ = 0**：I(m, r₁, 0) = 0（对所有 r₀ > 0）。
  **但借位情形 r₀ = 0, k₀ = 0** 产生额外贡献，以普通二项式 binom(r₁+k₁-1, r₁) 直接由商层处理（无需余数层 NTT）。

**商层**（固定 m, k₀）：

* **k₀ ≥ 1**：与前述相同：

$$J(m, k₀, k₁) = \frac{1}{k₁!} \sum\_{r₁} \frac{I(m, r₁, k₀)}{r₁!} \cdot (r₁+k₁)!$$

→ NTT 卷积：A'\[r₁] = I(m, r₁, k₀) / r₁!，翻转后与 B'\[t] = t! 卷积

* **k₀ = 0**（借位贡献）：对每个 r₁（≥ 0），所有余数集中在 r₀=0：

$$J\_0(m, k₁) = \frac{1}{k₁!} \sum\_{r₁} \frac{H\_{m, r₁d}}{r₁!} \cdot (r₁+k₁-1)!$$

→ NTT 卷积：A'\[r₁] = H\_{m, r₁d} / r₁!（仅 r₀=0 列），翻转后与 B⁺\[t] = (t-1)!（t ≥ 1, B⁺\[0]=0）卷积

**合并**：H\_{m+k,k} = b^m · a^{binom(k,2)} · J(m, k₀, k₁)，k₀ = 0 时使用 J\_0。总复杂度 O(N² log N)。

**失败用例**：

**根因分析**：检测得 a 的乘法阶 d=2（a ≡ MOD-1），触发四子路径中的"余数标量+商层标量"路径。此时 N/d = 1000，远超 NTT 阈值，但 d=2 < 64 → 余数层走标量；商层 k₁ 可达 ~1000，但遍历覆盖约 (N/d)² 范围。问题发生在余数层标量路径的内部求和阶段——纯标量路径使用 `ll sum = 0`（signed 64-bit `long long`）作为累加器。

1. 纯标量路径（q-Lucas 节末的 `else` 分支）
2. 余数-NTT + 商层标量路径
